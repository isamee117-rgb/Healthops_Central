Design — Section 1: Data Model

4 normalized tables replacing the 6 existing per-department form section tables

Table Structure

form_groups
id bigint PK
name varchar(100) — "IPD", "OPD", "Emergency", "OT"
context enum('ipd','opd','emergency','ot','general')
description text nullable
sort_order int default 0
is_active boolean default true
created_at / updated_at
forms
id bigint PK
form_group_id FK → form_groups
name varchar(150) — "Pre-Op Checklist"
description text nullable
is_active boolean default true
sort_order int default 0
created_at / updated_at
form_sections
id bigint PK
form_id FK → forms
title varchar(150) — "Patient Info"
sort_order int default 0
created_at / updated_at
form_components
id bigint PK
form_section_id FK → form_sections
type varchar(50) — 'text_input', 'checkbox'…
label varchar(200)
sort_order int default 0
config JSON — type-specific settings
created_at / updated_at

Relationships

FormGroup hasMany Form
Form       belongsTo FormGroup
Form       hasMany FormSection
FormSection belongsTo Form
FormSection hasMany FormComponent
FormComponent belongsTo FormSection

config JSON examples per component type

text_input
{ placeholder, required,
  maxlength, width('full'|'half'|'third') }
checkbox / radio
{ options:[{label,value}],
  required, layout('h'|'v') }
dynamic_table
{ columns:[{label,type,width}],
  min_rows:1, max_rows:50 }
header / signature / date
{ text, level('h2'|'h3') }
{ required, show_date:true }
{ required, width }

⚠ Migration Strategy — Existing Tables

These 6 tables will be replaced by the new system via a single migration:

opd_form_sections ipd_form_sections er_form_sections ot_form_sections ot_intraop_form_sections ot_postop_form_sections

A seeder will re-create the existing sections as Forms within the new structure. Existing API routes (/api/opd/form-sections etc.) will be updated to read from the new tables during Phase 1.
